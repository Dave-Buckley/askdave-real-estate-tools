**Comparison flow (from CONTEXT.md locked decisions):**

1. Agent is viewing a community detail. They tap "Compare with..." button.
2. Phase switches to `'compare'`. The current area is automatically the first column.
3. **Area picker sub-view** appears:
   - Top section: "Suggested similar areas" -- filter AREA_GUIDES where `priceTier` matches the current area AND at least one `propertyTypes` entry overlaps. If fewer than 3 matches, expand to adjacent tiers (e.g., premium also shows mid-premium). Exclude the current area.
   - Below: "All areas" list with the full AREA_GUIDES array (excluding current area), each with a checkbox/toggle to add to comparison.
   - Agent can select up to 2 more areas (3 total including the current one). Show a counter "2 of 3 selected" or similar.
   - "Compare" button at bottom (enabled when at least 1 additional area selected).
4. On "Compare" tap, show the **comparison table**:
   - Format: rows = metrics, columns = selected communities (up to 3)
   - Metrics to compare (each row):
     - Price/sqft (AED range)
     - Rental Yield (% range)
     - Service Charges (AED/sqft range)
     - Avg Transaction Price (AED range)
     - Price Growth YoY (%)
     - Freehold Status
     - Metro Access
     - Property Types
   - **Highlight logic per row:**
     - For yield, price growth: highest value gets green (#4ade80) text/bg tint, lowest gets red (#f87171) text/bg tint
     - For service charges: lowest is best (green), highest is worst (red)
     - For price/sqft: no best/worst highlight (depends on client budget), show neutral
     - For non-numeric rows (freehold, metro, property types): no highlighting
     - Use the midpoint of ranges for comparison (e.g., [1800, 2400] -> 2100)
   - Each metric cell shows the source + effective date in muted small text below the value
   - Community names as column headers with short names
   - Table should handle narrow panel width: use horizontal scroll if 3 columns don't fit, or use compact cell padding. Test readability at ~350px width.
   - "Back" button returns to the detail view of the area that initiated the comparison. Clear compareAreas state.

**WhatsApp share flow (from CONTEXT.md locked decisions):**

1. Agent is viewing a community detail. They tap "Share via WhatsApp" button.
2. A **share preview modal/overlay** appears (match existing TemplatePreview pattern from the app):
   - Editable textarea pre-filled with formatted area summary
   - Format:
     ```
     *{Community Name} - Area Guide*

     Price/sqft: AED {range}
     Gross Yield: {range}%
     Service Charges: AED {range}/sqft
     Freehold: {Yes/No}
     Metro: {Yes - station name / No}

     Data effective: {effectiveDate}
     Source: {source}
     {sourceUrl}
     ```
   - Agent can edit the text before sending (locked decision: must be able to verify/review)
   - "Send via WhatsApp" button calls `window.electronAPI.openExternal(\`https://wa.me/?text=\${encodeURIComponent(message)}\`)` (no-recipient share -- agent picks contact in WhatsApp)
   - "Cancel" button closes the preview
3. Use the existing dark theme styling for the overlay: `bg-[#161617]` card with `border border-white/[0.07]`, textarea with `bg-white/[0.06]`.

- Source attribution must appear in the comparison table cells (AREA-03 requirement applies to comparison view too).
