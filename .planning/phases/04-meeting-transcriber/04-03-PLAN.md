Build the desktop UI for the meeting transcriber with a source picker that supports two recording modes:

1. **Phone Mic (via WiFi)** -- QR code flow, phone records and sends audio over local network

Purpose: Completes the user-facing experience with flexible input sources -- agent can record from any available mic, not just phone.

**Desktop recording implementation:**

```typescript
// Start recording
const stream = await navigator.mediaDevices.getUserMedia({
  audio: { deviceId: { exact: selectedDeviceId } }
})

// Determine MIME type (same logic as phone recorder)
let mimeType = 'audio/webm;codecs=opus'
if (!MediaRecorder.isTypeSupported(mimeType)) mimeType = 'audio/webm'
if (!MediaRecorder.isTypeSupported(mimeType)) mimeType = 'audio/mp4'
if (!MediaRecorder.isTypeSupported(mimeType)) mimeType = ''

const recorder = new MediaRecorder(stream, mimeType ? { mimeType } : undefined)
const chunks: Blob[] = []
recorder.ondataavailable = (e) => { if (e.data.size > 0) chunks.push(e.data) }
recorder.start(1000)
```

**Stop + convert to Float32Array (16kHz mono):**
```typescript
// On stop: collect blob, convert to 16kHz mono Float32Array
const blob = new Blob(chunks, { type: recorder.mimeType || 'audio/webm' })
const arrayBuffer = await blob.arrayBuffer()
const audioCtx = new AudioContext()
const audioBuffer = await audioCtx.decodeAudioData(arrayBuffer)
audioCtx.close()

const targetRate = 16000
const numSamples = Math.ceil(audioBuffer.duration * targetRate)
const offlineCtx = new OfflineAudioContext(1, numSamples, targetRate)
const source = offlineCtx.createBufferSource()
source.buffer = audioBuffer
source.connect(offlineCtx.destination)
source.start(0)
const rendered = await offlineCtx.startRendering()
const pcmAudio = rendered.getChannelData(0) // Float32Array, 16kHz mono
```

After conversion, transition to shared transcription phase with the pcmAudio.

**Elapsed timer:** Use `setInterval` every 1 second. Track startTime, pausedDuration, pauseStart (same pattern as phone recorder). Use `tabular-nums` font feature.

**Cleanup on unmount or back:** Stop MediaRecorder, stop all stream tracks, clear interval.

## D) Shared Transcription Phase (phase = 'transcribing')

Entered from either phone flow (after getTranscriberAudio) or desktop flow (after stop+convert).

1. Create Whisper worker: `new Worker(new URL('../workers/whisper-worker.ts', import.meta.url), { type: 'module' })`
2. Get model ID from settings: `window.electronAPI.getSettings()` → `whisperModel`
3. Send `{ type: 'load', model }` to load the model
4. On 'loading' messages: show download progress bar if model needs downloading
5. On 'ready': send `{ type: 'transcribe', audio: pcmAudio }` with transferable `[pcmAudio.buffer]`
6. On 'result': store transcript in state, transition to 'done'
7. On 'error': show error state

**UI:**
- Spinner (`Loader2` with `animate-spin`) + "Transcribing..."
- If model downloading: "Downloading model (XX%)..." with progress bar
- Progress bar: `bg-[#161617]` track, `bg-[#818cf8]` fill, rounded, h-1.5
