<task type="auto">
  <name>Task 1: Create TranscriberView component with QR, status, and transcript display</name>
  <files>src/renderer/panel/components/TranscriberView.tsx</files>
  <action>
Create `src/renderer/panel/components/TranscriberView.tsx` -- the full transcriber UI component.

**Props:** `onBack: () => void` (called when agent navigates away; triggers cleanup)

**Component lifecycle:**
1. On mount: call `window.electronAPI.startTranscriberServer()` to start the local server. If it fails, show error state. If successful, get the `url` from the response.
2. Generate QR code data URL from the server URL using `QRCode.toDataURL(url, { width: 200, margin: 2, color: { dark: '#ededee', light: '#0d0d0e' } })`. Store in state.
3. Subscribe to `window.electronAPI.onTranscriberState(cb)` to receive live status updates from the server (phone connected, recording, paused, elapsed time, stopped).
4. On unmount (or when `onBack` is called): call `window.electronAPI.stopTranscriberServer()` to shut down the server and clean up. Remove the state listener.

**State machine (managed by listening to server state updates):**

```
idle -> (startServer) -> waiting -> (phone connects) -> connected/idle
  -> (phone starts recording) -> recording -> (phone pauses) -> paused
  -> (phone resumes) -> recording -> (phone stops) -> transcribing
  -> (whisper completes) -> done
```

**UI states (each renders different content):**

A) **Waiting for phone** (state = 'waiting', phoneConnected = false):
   - Large QR code image (centered)
   - Text below: "Scan this QR code on your phone"
   - Smaller text: "Make sure your phone and computer are on the same WiFi network"
   - A "Cancel" button at the bottom

B) **Phone connected, idle** (state = 'waiting', phoneConnected = true):
   - Green dot + "Phone connected"
   - Text: "Tap Start on your phone to begin recording"
   - The QR code is hidden now (phone already connected)
   - A "Disconnect" button

C) **Recording** (state = 'recording'):
   - Pulsing red dot indicator
   - "Recording..." text
   - Elapsed time display (MM:SS format, derived from `status.elapsed`)
   - Text: "Recording on phone... Tap Stop on your phone when done."
   - NO desktop controls for pause/stop (phone controls the recording per CONTEXT.md decision: "Agent can pause and resume recording from the phone browser interface" -- REC-02)

D) **Paused** (state = 'paused'):
   - Orange dot indicator
   - "Paused" text
   - Elapsed time frozen
   - "Tap Resume on your phone to continue"

E) **Transcribing** (state = 'transcribing'):
   - After phone sends audio, the component:
     1. Gets the audio via `window.electronAPI.getTranscriberAudio()`
     2. Creates the Whisper worker: `new Worker(new URL('../workers/whisper-worker.ts', import.meta.url), { type: 'module' })`
     3. Sends `{ type: 'load', model: settings.whisperModel }` to load the model
     4. On 'ready', sends `{ type: 'transcribe', audio }` with the Float32Array (use transferable: `[audio.buffer]`)
     5. Shows model loading progress if the model needs to download (first time)
   - UI: Spinner + "Transcribing..." or "Downloading model (XX%)..." during model load
   - Progress bar for model download if `modelLoading`

F) **Done** (state = 'done'):
   - "Transcript" heading
   - The transcript text in a scrollable, selectable container
   - Text should use `select-text` / `user-select: text` CSS so partial selection works
   - "Copy All" button -- copies full transcript to clipboard using `navigator.clipboard.writeText()`
   - Show a brief "Copied!" confirmation when copied (2 seconds)
   - "New Recording" button -- resets state, calls stopServer then startServer again
   - The transcript container should have a subtle border, monospace-ish font or regular font with good readability

G) **Error** (state = 'error'):
   - Error icon + error message
   - "Try Again" button (resets to idle)

**Styling guidelines (match existing app):**
- Dark theme: `bg-[#0d0d0e]`, `text-[#ededee]`, borders `border-white/[0.07]`
- Accent: indigo `text-[#818cf8]` for buttons/links, `bg-[rgba(99,102,241,0.14)]` for button backgrounds
- Use Lucide icons: `Mic` for recording indicator, `Loader2` with `animate-spin` for loading, `Copy` for copy button, `Check` for copied state, `Wifi`/`WifiOff` for connection status
- Component fills the available space (flex-1, overflow-y-auto for transcript)
- Recording timer uses `tabular-nums` font feature for stable width

**Whisper worker management:**
- Create the worker instance lazily (only when audio arrives, not on component mount)
- Terminate the worker on component unmount to free memory
- Store worker ref in useRef
- Get the model ID from settings: call `window.electronAPI.getSettings()` on mount, read `whisperModel`
- When worker sends `{ type: 'result', text }`, set the transcript in state, update server state to 'done'

**Ephemeral data (PRIV-01):**
- Transcript held only in React state -- navigating away unmounts the component and the data is gone
- Audio Float32Array is consumed once (getTranscriberAudio returns and nulls it on server side)
- Worker is terminated on unmount (frees model memory)

**Important implementation detail -- detecting when to start transcription:**
The onTranscriberState callback receives status updates from the server. When `state === 'transcribing'` (which the server sets when it receives the complete audio from the phone), the component should call `getTranscriberAudio()` to retrieve the audio, then feed it to the Whisper worker. This is the trigger for transcription.
