2. **Create `src/renderer/panel/lib/audio-recorder.ts`** -- a class that captures microphone audio as 16kHz mono PCM Float32Array, fully in-memory (no disk writes). Based on RESEARCH.md Pattern 2 but with these specifics:

- Constructor takes no args
- `async start(): Promise<void>` -- requests mic via `getUserMedia({ audio: { channelCount: 1, sampleRate: 16000, echoCancellation: true, noiseSuppression: true } })`, creates `AudioContext({ sampleRate: 16000 })`, connects `MediaStreamAudioSourceNode` to `ScriptProcessorNode(4096, 1, 1)`, captures PCM chunks into `Float32Array[]` array
- `pause(): void` -- sets `isPaused = true` (ScriptProcessor callback skips chunks)
- `resume(): void` -- sets `isPaused = false`
- `stop(): Float32Array` -- stops all tracks, disconnects nodes, closes AudioContext, merges all chunks into a single Float32Array, clears internal chunk array (release memory), returns the merged array
- `getElapsed(): number` -- returns total seconds of audio recorded (sum of chunk lengths / 16000)
- `isRecording(): boolean`, `isPaused(): boolean` getters
- Max recording cap: 60 minutes (3,600 seconds). The `onaudioprocess` callback should check elapsed and auto-stop if exceeded. Expose an `onMaxReached` callback property.
- No disk I/O anywhere. All data stays in memory as Float32Array.
- Export the class as default export.

2. **Create `src/renderer/panel/workers/whisper-worker.ts`** -- a Web Worker that handles Whisper model loading and audio transcription. Based on RESEARCH.md Pattern 1 and Pattern 5, with these specifics:

Message protocol (incoming `self.onmessage`):
- `{ type: 'load', model: WhisperModelId }` -- Load the specified Whisper model. Try WebGPU first (`navigator.gpu.requestAdapter()`), fall back to WASM on failure. Use `pipeline('automatic-speech-recognition', model, { device, dtype, progress_callback })`. Post `{ type: 'loading', progress }` during download, then `{ type: 'ready' }` when loaded. Use q8 quantization dtype: for WebGPU `{ encoder_model: 'fp32', decoder_model_merged: 'q8' }`, for WASM just `'q8'`.
- `{ type: 'transcribe', audio: Float32Array }` -- Transcribe the audio. Call `transcriber(audio, { chunk_length_s: 30, stride_length_s: 5, return_timestamps: false })`. Post `{ type: 'result', text: result.text }`.
- `{ type: 'unload' }` -- Release model from memory. Set transcriber to null. Post `{ type: 'unloaded' }`.

Message protocol (outgoing `self.postMessage`):
- `{ type: 'loading', progress: { status, file, loaded, total, progress } }`
- `{ type: 'ready' }`
- `{ type: 'result', text: string }`
- `{ type: 'error', error: string }`
- `{ type: 'unloaded' }`

Handle errors with try/catch in both load and transcribe, posting `{ type: 'error', error: message }`.

IMPORTANT: The worker uses Vite's `import.meta.url` pattern. Configure `env.allowRemoteModels = true` for model downloads from HuggingFace Hub. Do NOT set `env.cacheDir` -- let the browser Cache API handle caching automatically (this is the correct behavior in Electron's renderer/worker context per RESEARCH.md Pitfall 5).
