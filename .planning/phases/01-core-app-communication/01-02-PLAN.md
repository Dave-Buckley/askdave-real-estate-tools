<task type="auto">
  <name>Task 1: Phone normalization, clipboard polling, tray + window management</name>
  <files>src/main/phone.ts, src/main/clipboard.ts, src/main/tray.ts</files>
  <action>
    1. **src/main/phone.ts** — Phone number normalization module (APP-04):
       - `normalizeToUAE(raw: string): string | null` — uses `parsePhoneNumber(raw.trim(), 'AE')` from `libphonenumber-js`. Returns E.164 string (`+971XXXXXXXXX`) if valid, `null` if not. Wrap in try/catch.
       - `formatForDisplay(e164: string): string` — uses `parsePhoneNumber(e164)` and calls `.formatInternational()` to produce readable format like "+971 50 123 4567".
       - Export both functions.

    2. **src/main/clipboard.ts** — Clipboard polling service (APP-02):
       - UAE loose regex: `/(?:(?:\+|00)971|0)(?:5[0-9]|[2-9])\d{7}/g` — catches raw clipboard text containing UAE numbers.
       - `startClipboardPolling(onPhoneDetected: (e164: string, displayNumber: string) => void)` — uses `setInterval` at **500ms** (not faster — per RESEARCH.md anti-pattern). On each tick:
         a. Read `clipboard.readText()`
         b. Compare to `lastClipboardText` — skip if unchanged
         c. Match against UAE regex
         d. For first match, call `normalizeToUAE()` from phone.ts
         e. If valid, call `formatForDisplay()` and invoke callback with both E.164 and display format
       - `stopClipboardPolling()` — clears interval.
       - Track `lastClipboardText` to avoid duplicate detections.
       - Export both functions.

    3. **src/main/tray.ts** — Tray icon + window management:
       - `createTray()` — creates `Tray` with app icon, sets tooltip "Agent Kit". Returns tray instance.
       - `createTrayPanel(tray: Tray): BrowserWindow` — creates frameless, non-resizable, alwaysOnTop BrowserWindow (360x480). Loads `src/renderer/panel/index.html`. Uses preload `src/preload/index.ts`. On tray click, toggles panel visibility.
       - `positionPanel(tray, win)` — positions panel near tray using `tray.getBounds()` and `screen.getDisplayNearestPoint()`. Above tray on Windows (taskbar bottom), below menu bar on macOS (per RESEARCH.md Pattern 1). Account for DPI scaling on Windows per RESEARCH.md Pitfall 4.
       - `createPopupWindow(tray: Tray): BrowserWindow` — creates small frameless, transparent, alwaysOnTop window (320x60). Loads popup HTML. Uses preload `src/preload/popup.ts`.
       - `showPopup(e164: string, displayNumber: string)` — positions popup near tray and sends `popup:show` event with phone data. Shows window.
       - `hidePopup()` — hides popup window.
       - Panel hides when it loses focus (`win.on('blur', () => win.hide())`).
       - Context menu on tray right-click: "Open Panel", "Settings", separator, "Quit".
       - Export all functions.
  </action>

<task type="auto">
  <name>Task 2: Phone actions, hotkeys, updater, IPC handlers, preload bridges, and app bootstrap</name>
  <files>src/main/actions.ts, src/main/hotkeys.ts, src/main/updater.ts, src/main/ipc.ts, src/main/index.ts, src/preload/index.ts, src/preload/popup.ts, src/preload/settings.ts</files>
  <action>
    1. **src/main/actions.ts** — Phone actions (COMM-01, COMM-02):
       - `openDialler(e164: string)` — calls `shell.openExternal(\`tel:${e164}\`)`. Wrap in try/catch, log errors.
       - `openWhatsApp(e164: string, mode: 'web' | 'desktop')` — strips '+' prefix. If mode 'web': `https://wa.me/{digits}`. If mode 'desktop': `whatsapp://send?phone={digits}`. Uses `shell.openExternal`. Wrap in try/catch.
       - `buildWhatsAppURL(e164: string, message: string): string` — builds `https://wa.me/{digits}?text={encoded_message}` for template sending (COMM-03 support).
       - Export all functions.

    2. **src/main/hotkeys.ts** — Global hotkey registration (APP-01):
       - `registerHotkeys(config: HotkeyConfig, getActiveNumber: () => string | null, whatsappMode: 'web' | 'desktop')` — registers shortcuts after `app.whenReady()`. Checks return value of `globalShortcut.register()` — if `false`, log warning (per RESEARCH.md Pitfall 6: "This hotkey is already in use"). When triggered, gets active number from callback; if no number active, do nothing (no error).
       - `unregisterHotkeys()` — calls `globalShortcut.unregisterAll()`. Called on `app.on('will-quit')`.
       - Export both functions.

    3. **src/main/updater.ts** — Auto-update (APP-05):
       - `setupAutoUpdater()` — imports `autoUpdater` from `electron-updater`. Sets `autoInstallOnAppQuit = true`. Calls `autoUpdater.checkForUpdatesAndNotify()`. Wraps in try/catch (fails gracefully in dev mode where no update config exists).
       - Export function.

    4. **src/main/ipc.ts** — IPC handler registration:
       Register all `ipcMain.handle` and `ipcMain.on` handlers:

       **One-way (renderer → main):**
       - `action:dial` — calls `openDialler(e164)`
       - `action:whatsapp` — calls `openWhatsApp(e164, mode)`
       - `action:whatsapp-with-message` — calls `shell.openExternal(buildWhatsAppURL(e164, message))`
       - `popup:dismiss` — calls `hidePopup()`
       - `popup:dial` — calls `openDialler(e164)`
       - `popup:whatsapp` — calls `openWhatsApp(e164, mode)` using stored preference

       **Two-way (renderer → main → renderer):**
       - `store:getSettings` — returns `store.store` (full settings object)
       - `store:saveSettings` — receives partial settings, calls `store.set()` for each key
       - `store:getTemplates` — returns `store.get('templates')`
       - `store:saveTemplates` — sets `store.set('templates', templates)`
       - `store:addTemplate` — adds template to array, returns updated array
       - `store:deleteTemplate` — removes by id, returns updated array
       - `store:updateTemplate` — updates by id, returns updated array

       Accept tray/popup window references so handlers can call showPopup/hidePopup.

    5. **src/preload/index.ts** — Panel preload (contextBridge):
       ```typescript
       contextBridge.exposeInMainWorld('electronAPI', {
         // Actions
         dial: (e164: string) => ipcRenderer.send('action:dial', e164),
         openWhatsApp: (e164: string, mode: 'web' | 'desktop') => ipcRenderer.send('action:whatsapp', e164, mode),
         sendWhatsAppMessage: (e164: string, message: string) => ipcRenderer.send('action:whatsapp-with-message', e164, message),

         // Store
         getSettings: () => ipcRenderer.invoke('store:getSettings'),
         saveSettings: (settings: Partial<AppSettings>) => ipcRenderer.invoke('store:saveSettings', settings),
         getTemplates: () => ipcRenderer.invoke('store:getTemplates'),
         saveTemplates: (templates: Template[]) => ipcRenderer.invoke('store:saveTemplates', templates),
         addTemplate: (template: Template) => ipcRenderer.invoke('store:addTemplate', template),
         deleteTemplate: (id: string) => ipcRenderer.invoke('store:deleteTemplate', id),
         updateTemplate: (template: Template) => ipcRenderer.invoke('store:updateTemplate', template),

         // Events from main
         onPhoneDetected: (cb: (e164: string, displayNumber: string) => void) =>
           ipcRenderer.on('phone:detected', (_e, e164, displayNumber) => cb(e164, displayNumber)),
         removePhoneDetectedListener: () => ipcRenderer.removeAllListeners('phone:detected'),
       })
       ```

    6. **src/preload/popup.ts** — Popup preload:
       ```typescript
       contextBridge.exposeInMainWorld('popupAPI', {
         dial: (e164: string) => ipcRenderer.send('popup:dial', e164),
         openWhatsApp: (e164: string) => ipcRenderer.send('popup:whatsapp', e164),
         dismiss: () => ipcRenderer.send('popup:dismiss'),
         onShow: (cb: (e164: string, displayNumber: string) => void) =>
           ipcRenderer.on('popup:show', (_e, e164, displayNumber) => cb(e164, displayNumber)),
       })
       ```

    7. **src/preload/settings.ts** — Settings preload:
       ```typescript

       })
       ```

    8. **src/main/index.ts** — App bootstrap (wire everything together):
       - Import all modules
       - `app.whenReady()`:
         a. Create tray via `createTray()`
         b. Create panel window via `createTrayPanel(tray)`
         c. Create popup window via `createPopupWindow(tray)`
         d. Register IPC handlers via `registerIPCHandlers(store, popupWindow, ...)`
         e. Read settings from store, build HotkeyConfig, register hotkeys
         f. If `clipboardEnabled`, start clipboard polling with callback that calls `showPopup` and sends `phone:detected` to panel
         g. Set login item: `app.setLoginItemSettings({ openAtLogin: true, ...(process.platform === 'darwin' ? { type: 'mainAppService' } : {}) })` (APP-05)
         h. Call `setupAutoUpdater()` (APP-05)
         i. Register store change listeners for hotkey re-registration — so settings changes take effect immediately without an app restart:
            ```typescript
            store.onDidChange('dialHotkey', () => {
              unregisterHotkeys()
              registerHotkeys(buildHotkeyConfig(store), getActiveNumber, store.get('whatsappMode'))
            })
            store.onDidChange('whatsappHotkey', () => {
              unregisterHotkeys()
              registerHotkeys(buildHotkeyConfig(store), getActiveNumber, store.get('whatsappMode'))
            })
            store.onDidChange('hotkeysEnabled', () => {
              unregisterHotkeys()
              if (store.get('hotkeysEnabled')) {
                registerHotkeys(buildHotkeyConfig(store), getActiveNumber, store.get('whatsappMode'))
              }
            })
            ```
            Where `buildHotkeyConfig(store): HotkeyConfig` is a small helper that reads `dialHotkey`, `whatsappHotkey`, and `hotkeysEnabled` from the store.
       - `app.on('will-quit')`: unregister hotkeys, stop clipboard polling
       - `app.on('window-all-closed')`: prevent app quit (tray app stays running)
       - Handle Settings menu item: create/show settings window
