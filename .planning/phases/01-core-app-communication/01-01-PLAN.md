<task type="auto">
  <name>Task 2: Define shared types and electron-store schema with default templates</name>
  <files>src/shared/types.ts, src/main/store.ts</files>
  <action>
    1. Create `src/shared/types.ts` with the following interfaces:

       ```typescript
       export interface Template {
         id: string
         name: string        // Display name (e.g., "Listing Introduction")
         body: string        // Message body with {name} placeholder
         category: string    // e.g., "introduction", "follow-up", "viewing"
       }

       export interface AppSettings {
         clipboardEnabled: boolean
         hotkeysEnabled: boolean
         dialHotkey: string           // Electron accelerator format, e.g., "CommandOrControl+Shift+D"
         whatsappHotkey: string
         whatsappMode: 'web' | 'desktop'
         templates: Template[]
       }

       export interface HotkeyConfig {
         dialEnabled: boolean
         dialKey: string
         whatsappEnabled: boolean
         whatsappKey: string
       }

       export interface ContactInfo {
         e164: string         // Normalized phone number (+971XXXXXXXXX)
         displayNumber: string // Formatted for display (e.g., "+971 50 123 4567")
         name?: string        // Optional contact name for template substitution
       }
       ```

    2. Create `src/main/store.ts` with electron-store instance:

       - Import `Store` from `electron-store` (use `require` if v9, `import` if v11)
       - Define `DEFAULT_TEMPLATES` array with 6 UAE real estate templates:
         a. "Listing Introduction" — agent introducing a new property listing
         b. "Follow-Up" — checking in after a viewing or call
         c. "Viewing Confirmation" — confirming a scheduled viewing
         d. "Viewing Reminder" — reminding about upcoming viewing
         e. "New Listing Alert" — notifying a client about a matching property
         f. "Thank You" — post-viewing or post-meeting thank you

       Each template body should be realistic UAE real estate text with `{name}` placeholder. Example:
       ```
       "Hi {name}, this is [Agent] from [Agency]. I wanted to follow up on our conversation about the property. Would you like to schedule a viewing? Let me know what time works best for you."
       ```
       Use `{name}` for the client name. Leave agent/agency as literal text (users will edit these to personalize).

       - Create store with `defaults` matching `AppSettings` interface:
         - `clipboardEnabled: true`
         - `hotkeysEnabled: true`
         - `dialHotkey: 'CommandOrControl+Shift+D'`
         - `whatsappHotkey: 'CommandOrControl+Shift+W'`
         - `whatsappMode: 'web'` (safe default per RESEARCH.md — always works)
         - `templates: DEFAULT_TEMPLATES`

       - Export the store instance and DEFAULT_TEMPLATES

    3. Generate unique IDs for default templates using a simple approach: `crypto.randomUUID()` or a short ID like `tpl-001` through `tpl-006`.
  </action>
