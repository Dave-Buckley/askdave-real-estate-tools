**2. Completely rewrite `src/main/onenote.ts`:**

Delete all existing code (PowerShell COM approach). Replace with Graph API implementation:

**a. Imports and ROLE_TEMPLATES:**
- Import `{ Client }` from `@microsoft/microsoft-graph-client` (already in package.json v3.0.7)
- Import `{ shell }` from `electron`
- Import `microsoftGetAccessToken` from `./auth/microsoft`
- Import `store` from `./store`
- Import `ContactRole` from `../shared/types`
- Import `upsertContact` from `./contacts`
- Keep the exact same `ROLE_TEMPLATES` constant (copy from existing code — same structure, same questions). Export it for potential reuse.

**b. `getGraphClient()` function:**
Create a Graph API client using custom auth provider pattern:
```typescript
function getGraphClient(): Client {
  return Client.initWithMiddleware({
    authProvider: {
      getAccessToken: async () => {
        const token = await microsoftGetAccessToken()
        if (!token) throw new Error('Microsoft not signed in — please sign in via Settings')
        return token
      }
    }
  })
}
```

**c. `ensureNotebookAndSection(client)` function:**
- Check `store.get('oneNoteNotebookId')` and `store.get('oneNoteSectionId')` — if both cached, return them immediately.
- Otherwise, GET `/me/onenote/notebooks` with `.filter("displayName eq 'Real Estate'")` and `.select('id,displayName')`.
- If notebook found, use its `id`. If not found, POST to `/me/onenote/notebooks` with `{ displayName: 'Real Estate' }`.
- Then GET `/me/onenote/notebooks/{notebookId}/sections` with `.filter("displayName eq 'Contacts'")` and `.select('id,displayName')`.
- If section found, use its `id`. If not found, POST to `/me/onenote/notebooks/{notebookId}/sections` with `{ displayName: 'Contacts' }`.
- Cache both IDs: `store.set('oneNoteNotebookId', notebookId)` and `store.set('oneNoteSectionId', sectionId)`.
- Return `{ notebookId, sectionId }`.

**d. `buildPageHTML(e164, name, displayNumber, roles)` function:**
Build the full HTML page content for Graph API. The page title MUST be the E.164 number (this is the lookup key). Structure:
```html
<!DOCTYPE html>
<html>
  <head>
    <title>{e164}</title>
    <meta name="created" content="{ISO date}" />
  </head>
  <body>
    <h1>{name || e164}</h1>
    <p><b>Phone:</b> {displayNumber}</p>
    <p><b>Roles:</b> {roles.join(', ')}</p>
    {for each role: buildRoleSectionHTML(role)}
  </body>
</html>
```

**e. `buildRoleSectionHTML(role)` function:**
Build ONE role's qualifying template as HTML:
```html
<h2>{ROLE_TEMPLATES[role].label}</h2>
<table border="1">
  <tr><th>Question</th><th>Answer</th></tr>
  {for each question: <tr><td>{question}</td><td></td></tr>}
</table>
```

**f. `findPageByPhone(client, sectionId, e164)` function:**
- GET `/me/onenote/sections/{sectionId}/pages` with `.filter("title eq '${e164}'")` and `.select('id,title,links')`.
- NOTE: The `+` in E.164 must be URL-encoded. The Graph SDK's `.filter()` handles this, but be aware.
- Return `page` object (with `id` and `links.oneNoteClientUrl.href`) if found, `null` if not.

**g. `openContactPage(data)` — the main exported function:**
Signature: `export async function openContactPage(data: { name: string; displayNumber: string; roles: ContactRole[]; e164: string }): Promise<{ success: boolean; error?: string; pageId?: string }>`

Steps:
1. Call `getGraphClient()`.
2. Call `ensureNotebookAndSection(client)` to get `sectionId`.
3. Call `findPageByPhone(client, sectionId, data.e164)`.
4. **If page exists:**
   - Check if contact has NEW roles not yet on the page. Compare `data.roles` with stored contact's roles from a previous `openContactPage` call. The way to detect "new role just added" is: check the existing `Contact` record via `store.get('contacts')[data.e164]`, look at its `oneNotePageId` — if the page already exists AND the contact now has roles that were not there when the page was last fully built, call `appendRoleSection()` for each new role.
   - Open the page: `await shell.openExternal(page.links.oneNoteClientUrl.href)`
5. **If page does NOT exist:**
   - Use `data.roles` (default to `['Tenant']` if empty, per existing behavior).
   - POST to `/me/onenote/sections/{sectionId}/pages` with `Content-Type: text/html` and the full HTML from `buildPageHTML()`.
   - Extract `response.id` and `response.links.oneNoteClientUrl.href` from the created page.
   - Update the contact record: `upsertContact(data.e164, { oneNotePageId: response.id })`.
   - Open the page: `await shell.openExternal(response.links.oneNoteClientUrl.href)`
6. Return `{ success: true, pageId }`.
7. Wrap everything in try/catch. On auth error, return `{ success: false, error: 'Microsoft not signed in — please sign in via Settings' }`. On other Graph errors, return `{ success: false, error: message }`.

**h. `appendRoleSection(client, pageId, role)` function:**
- Build the role HTML using `buildRoleSectionHTML(role)`.
- PATCH to `/me/onenote/pages/{pageId}/content` with `Content-Type: application/json` body:
  ```json
  [{ "target": "body", "action": "append", "content": "<role section HTML>" }]
  ```
- Use `JSON.stringify()` for the PATCH body.

**Important anti-patterns to avoid (from RESEARCH.md):**
- Do NOT use `?sectionName=` parameter — always use explicit section ID endpoint
- Do NOT use `replace` action for body — use `append`
- Do NOT hardcode notebook/section IDs — always find-or-create then cache
- Do NOT skip auth check — always verify `microsoftGetAccessToken()` returns non-null before proceeding

**3. Update store defaults in `src/main/store.ts`:**
No change needed to defaults (the new fields are optional and undefined by default in electron-store). But verify that `store.get('oneNoteNotebookId')` returns `undefined` gracefully (it will, since electron-store returns `undefined` for missing keys).
