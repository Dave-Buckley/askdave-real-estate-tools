**File 1: `src/renderer/panel/components/MortgageCalc.tsx`** -- Most complex calculator.

Props: `{ onClear: (clearFn: () => void) => void, onShare: (text: string) => void }`

Inputs (all text fields, controlled state):
- Property Price (AED) -- empty default, placeholder "e.g., 2,000,000"
- Interest Rate (%) -- default from CALCULATOR_RATES.mortgage.defaultInterestRate.value ("4.99")
- Term (years) -- default from CALCULATOR_RATES.mortgage.maxTermYears.value ("25")
- Two toggles (per research discretion recommendation):
  - Resident / Non-resident toggle (styled as segmented control with `bg-[#818cf8]/20` active)
  - First Property / 2nd+ Property toggle (greyed out and disabled when Non-resident selected, since non-residents always get 50% LTV)
- Down Payment (%) -- auto-populated from LTV rule, editable. Track whether user has manually overridden.

LTV logic using `CALCULATOR_RATES.mortgage.ltvRules`:
- Look up the correct LtvEntry based on toggle state + property price
- Auto-set down payment to `ltvRule.minDown * 100` UNLESS:
  - User has manually typed a higher down payment (track with `isDownPaymentOverridden` boolean)
  - If user's custom down payment is HIGHER than the new minimum, keep it
  - If user's custom down payment is LOWER than the new minimum, override it (minimum required)
- Show the active LTV rule label below the toggles (e.g., "Resident, 1st property <=5M: max 80% LTV")

PMT calculation via `useMemo`:
```
monthlyRate = annualRate / 100 / 12
totalPayments = termYears * 12
principal = propertyPrice - downPaymentAmount
if (monthlyRate === 0) monthlyPayment = principal / totalPayments
else factor = (1 + monthlyRate)^totalPayments; monthlyPayment = principal * (monthlyRate * factor) / (factor - 1)
totalRepayment = monthlyPayment * totalPayments
totalInterest = totalRepayment - principal
```

Guard: if propertyPrice is 0 or empty, return null. Show placeholder.

Results display:
- "Down Payment" line: formatAED + "(XX%)" label
- "Loan Amount" line: formatAED
- "Monthly Payment" line: formatAEDDecimal (bold, larger text -- this is the headline number)
- "Total Repayment" line: formatAED
- "Total Interest" line: formatAED
- "LTV Applied" line: show the percentage and rule label

Rate attribution: Inline next to Interest Rate input -- source name + effective date + freshness dot + ExternalLink icon. When agent overrides rate, show "Custom value (default: 4.99%)". Same pattern for LTV rule source attribution.

"Share via WhatsApp" button at the bottom of results that calls `onShare(buildMortgageSummary())`.

- Agency Fee = propertyPrice * (agencyFeePercent / 100)

- Total to Close = propertyPrice + totalCosts
